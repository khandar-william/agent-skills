# Dynamic Outline Pattern (Adapted from WebWeaver Planner)

## Outline Rules

### Structure Requirements
- Hierarchical numbering: I. / A. / 1. / a. (up to 4 levels)
- Only section/subsection TITLES in the outline (not full content)
- Each subsection must have source citations: `[1, 3, 7]`
- Logical flow: context → analysis → implications → conclusion

### Iterative Refinement
- Update the outline at least **3 times** during research
- Each round: search → discover → restructure
- Merge subsections with similar content (combine citations)
- Add subsections for gaps discovered
- Remove subsections that prove irrelevant
- Ensure every subsection has at least one citation

### Quality Checks
Before finalizing the outline:
1. Does every subsection have citations? → If not, search for that specific topic
2. Are there 3+ major sections? → If not, search for more facets
3. Is the logical flow clear? → Reorder if needed
4. Are there duplicate subsections? → Merge them
5. Does the outline cover: background, analysis, evidence, implications? → Fill gaps

### Outline Template

```markdown
# [Working Title]

## I. [Background/Context]
### A. [Historical context / definitions] [1, 2]
### B. [Current landscape / state of the art] [3, 4, 5]
#### 1. [Specific subtopic] [3]
#### 2. [Specific subtopic] [4, 5]

## II. [Core Analysis / Main Findings]
### A. [Key finding / theme 1] [6, 7, 8]
#### 1. [Supporting evidence] [6, 7]
#### 2. [Analysis / implications] [8]
### B. [Key finding / theme 2] [9, 10]
#### 1. [Supporting evidence] [9]
#### 2. [Analysis / implications] [10]
### C. [Comparison / contrast if applicable] [6, 9, 11]

## III. [Implications / Applications]
### A. [Practical implications] [12, 13]
### B. [Limitations / challenges] [14, 15]
### C. [Future directions] [16, 17]

## IV. [Conclusion / Synthesis]
### A. [Key takeaways] [all]
### B. [Answer to original question] [all]
```

### Source ID Tracking
As you discover sources, maintain a running ID counter:
- Source 1: First page visited
- Source 2: Second page visited
- etc.
- Replace URLs in the outline with [ID] references
- Keep the full Memory Bank for the writer phase
